# 处罚模块使用文档（给新手）

> 适用对象：完全不懂代码、只想会用命令的管理员。  
> 核心命令：`/处罚`

---

## 1. 你先知道这几件事

1. **只有有管理权限的人才能用**（模块内部会做管理员权限检查）。
2. 处罚命令的回复默认是**仅你可见**（ephemeral）。
3. 处罚相关的“同步”参数，**默认是开启**的。  
   - 也就是你不填 `同步` 时，系统会按“已配置的同步服务器”尝试跨服同步。
4. 跨服同步关系是**单向**：A 服配置 B，只代表 `A → B`。  
   双向要在 B 服再配置一次 A（`B → A`）。

---

## 2. 开始前的准备（建议按顺序）

### 第一步：配置公告频道（可选，但强烈建议）
用于发送处罚公告。支持多个频道，也支持跨服务器频道 ID。

### 第二步：配置警告身份组（如果你要用“警告”功能）
没有配置警告身份组时：
- `/处罚 警告` 会直接报错
- `/处罚 禁言` 里的“警告时长”会无法附加警告身份组

### 第三步：配置同步服务器（可选）
如果你希望处罚跨服务器自动同步，需要先配置同步目标。

---

## 3. 时间格式怎么填（很重要）

适用于 `时长`、`警告时长` 这类参数：

- `12h` = 12 小时
- `3d` = 3 天
- `7` = 7 天（纯数字默认按天）

注意：
- 不能填 0 或负数
- 禁言最长 28 天（Discord 限制）

---

`/处罚` 下有 9 个子命令：

1. 封禁

下面逐个讲。

## 5. 子命令详细说明（每个参数都写清楚）

---

## 5.1 `/处罚 封禁`
永久封禁成员（会把成员移出服务器并禁止再次加入，直到解封）。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要封禁的目标用户 |
| 原因 | 文本 | 否 | 未说明 | 封禁原因 |
| 同步 | 布尔(true/false) | 否 | `true` | 是否同步到已配置的关联服务器 |

示例：
- `/处罚 封禁 用户:@小明 原因:恶意广告`
- `/处罚 封禁 用户:@小明 原因:恶意广告 同步:false`

---

## 5.2 `/处罚 解封`
解除对某个用户 ID 的封禁。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 用户id | 文本 | 是 | 无 | 被解封用户的 Discord ID（不是 @提及） |
| 原因 | 文本 | 否 | 未说明 | 解封原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：
- `/处罚 解封 用户id:123456789012345678 原因:申诉通过`

---

## 5.3 `/处罚 禁言`
对成员执行超时禁言（Timeout）。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要禁言的成员 |
| 时长 | 文本 | 是 | 无 | 如 `2h`、`3d`、`7` |
| 原因 | 文本 | 否 | 未说明 | 禁言原因 |
| 警告时长 | 文本 | 否 | 不添加警告身份组 | 填了就会尝试同时加“警告身份组”，如 `7d` |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：
- `/处罚 禁言 用户:@小明 时长:3d 原因:刷屏`
- `/处罚 禁言 用户:@小明 时长:12h 警告时长:7d 原因:辱骂`

补充说明：
- 禁言超过 28 天会失败
- 如果填了“警告时长”但本服没配置警告身份组，则不会成功附加警告身份组

---

## 5.4 `/处罚 警告`
给成员添加“警告身份组”，到期自动移除。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 目标成员 |
| 时长 | 文本 | 是 | 无 | 如 `7`、`12h`、`3d` |
| 原因 | 文本 | 否 | 未说明 | 警告原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：
- `/处罚 警告 用户:@小明 时长:7d 原因:轻度违规`

注意：
- 先配置警告身份组，否则该命令会报错。

---

## 5.5 `/处罚 解除禁言`
解除成员的禁言。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要解除禁言的成员 |
| 原因 | 文本 | 否 | 未说明 | 解除原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：
- `/处罚 解除禁言 用户:@小明 原因:已改正`

---

## 5.6 `/处罚 配置公告频道`
管理处罚公告发送到哪些频道。支持多频道、跨服务器频道 ID。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 操作 | 选择(add/remove/list) | 是 | 无 | 添加 / 移除 / 查看列表 |
| 频道 | 频道 | 否 | 无 | 当前服务器内可直接选择 |
| 频道id | 文本 | 否 | 无 | 跨服务器频道请填 ID |

参数规则：
1. `list`：只看列表，不需要填频道。
2. `add/remove`：`频道` 和 `频道id` 至少填一个。
3. 如果两个都填，系统优先用 `频道id`。

示例：
- `/处罚 配置公告频道 操作:list`
- `/处罚 配置公告频道 操作:add 频道:#处罚公告`
- `/处罚 配置公告频道 操作:add 频道id:123456789012345678`
- `/处罚 配置公告频道 操作:remove 频道id:123456789012345678`

---

## 5.7 `/处罚 配置警告身份组`
设置本服务器用于“警告处罚”的身份组。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 身份组 | 身份组 | 是 | 无 | 例如 @警告组 |

示例：
- `/处罚 配置警告身份组 身份组:@警告组`

---

## 5.8 `/处罚 配置同步服务器`
管理处罚同步目标服务器（**单向配置**）。

| 参数 | 类型 | 必填 | 默认值 | 用法说明 |
|---|---|---|---|---|
| 操作 | 选择(add/remove/list) | 是 | 无 | 添加 / 移除 / 查看列表 |
| 目标服务器id | 文本 | 否 | 无 | add/remove 时必填；list 不填 |

示例：
- `/处罚 配置同步服务器 操作:list`
- `/处罚 配置同步服务器 操作:add 目标服务器id:987654321098765432`
- `/处罚 配置同步服务器 操作:remove 目标服务器id:987654321098765432`

重点：
- A 服加 B，只是 `A → B`
- 想双向同步，B 服还要再加 A（`B → A`）

---

|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要查询的用户 |

示例：

## 6. 公告消息会显示什么？

当前公告是卡片式 Embed，大体会显示：

- 成员、管理员、原因
- 时长（适用于禁言/警告）
- 处罚范围（本次涉及的服务器）
- 处罚ID
- 撤销处罚时会显示原处罚类型、原处罚ID（找不到也不影响执行）

颜色规则：
- 禁言、警告：橙色
- 封禁：红色
- 撤销类（解封、解除禁言）：粉紫色

## 7. 常见问题（FAQ）

### Q1：提示“你没有权限使用此命令”
你不在管理员权限名单中，联系服主检查权限配置。

### Q2：提示“无法找到该用户”
目标用户可能不在当前服务器，或你填错了对象。

### Q3：跨服同步失败
常见原因：
- 机器人不在目标服务器
- 机器人权限不足（无封禁/禁言/管理身份组权限）
- 目标用户不在目标服务器（禁言/警告/解除禁言需要用户在服）

### Q4：配置公告频道失败
常见原因：
- 填错频道 ID
- 机器人看不到该频道
- 频道不是文本频道

### Q5：警告命令失败
通常是没有先执行 `/处罚 配置警告身份组`。

## 8. 给新管理员的一套推荐流程

1. `/处罚 配置公告频道 操作:add 频道:#处罚公告`
2. `/处罚 配置警告身份组 身份组:@警告组`
3. （可选）`/处罚 配置同步服务器 操作:add 目标服务器id:...`
4. 正式使用处罚命令（封禁/禁言/警告等）
5. 通过 `/处罚 查询记录` 回看历史
