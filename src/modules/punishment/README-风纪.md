# 风纪指令使用文档（给新手）

> 适用对象：各板块的**风纪委员**，以及负责给他们开权限的**管理员**。
> 核心命令：`/风纪`（风纪委员用）、`/风纪配置`（管理员用）

---

## TL;DR（一句话看懂）

- **`/风纪` 是 `/处罚` 的受限版**：只能 `禁言`、`警告`、`解除禁言`、`查询记录`，**不能封禁/解封**，且禁言、警告时长有**上限**。
- **谁能用**：管理员用 `/风纪配置 身份组 add @某身份组` 授权；没被授权的人用不了（管理员本身一直能用）。
- **管上限**：管理员用 `/风纪配置 上限 禁言上限:2h 警告上限:3d` 设定风纪能开的最长时长，超了会被拒绝。
- **配置共用**：警告身份组、公告频道、跨服同步，全部**和 `/处罚` 用同一套**，在 `/处罚` 那边配一次即可，风纪这边不用重复配。
- **默认上限**：管理员没设时，禁言和警告默认上限都是 **28 天**。

管理员三步开通：
1. `/风纪配置 身份组 add 身份组:@风纪委员`
2. `/风纪配置 上限 禁言上限:2h 警告上限:7d`
3. 让风纪委员用 `/风纪 禁言 / 警告 / 解除禁言`

---

## 1. 它和 `/处罚` 有什么区别？

| | `/处罚`（管理组） | `/风纪`（风纪委员） |
|---|---|---|
| 谁能用 | 管理员权限名单 | 管理员 **+** 被授权的风纪身份组 |
| 封禁 / 解封 | ✅ 有 | ❌ 没有 |
| 禁言 / 警告 | ✅ 无额外上限（禁言仍受 Discord 28 天限制） | ✅ 有，**受管理员设定的上限约束** |
| 解除禁言 / 查询记录 | ✅ | ✅ |
| 配置类子命令 | ✅ 在 `/处罚` 里 | ❌ 用 `/风纪配置`（仅管理员） |
| 警告组/公告/同步配置 | 在这里配 | **共用 `/处罚` 的配置** |

一句话：风纪委员能在授权范围内管本板块成员，但摸不到“封禁”和“配置”这类高权限操作。

## 2. 风纪委员用的命令 `/风纪`

> 前提：你的身份组已被管理员加入风纪名单，否则会提示权限不足。
> 回复默认**仅你可见**；`同步` 默认开启（同 `/处罚`）。

### 2.1 `/风纪 禁言`
对成员超时禁言，**时长不能超过管理员设的“禁言上限”**。

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要禁言的成员 |
| 时长 | 文本 | 是 | 无 | 如 `2h`、`3d`、`7`（纯数字=天） |
| 原因 | 文本 | 否 | 未说明 | 禁言原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：`/风纪 禁言 用户:@小明 时长:2h 原因:刷屏`
超上限时会提示“禁言时长超过上限，风纪禁言最长为 XX”。

### 2.2 `/风纪 警告`
给成员加“警告身份组”（到期自动移除），**时长不能超过“警告上限”**。

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 目标成员 |
| 时长 | 文本 | 是 | 无 | 如 `7`、`12h`、`3d` |
| 原因 | 文本 | 否 | 未说明 | 警告原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：`/风纪 警告 用户:@小明 时长:3d 原因:轻度违规`
注意：需要管理员先在 `/处罚 配置警告身份组` 设好警告身份组，否则会失败。

### 2.3 `/风纪 解除禁言`
解除成员禁言（若持有警告身份组会一并移除）。

| 参数 | 类型 | 必填 | 默认 | 说明 |
|---|---|---|---|---|
| 用户 | 用户 | 是 | 无 | 要解除禁言的成员 |
| 原因 | 文本 | 否 | 未说明 | 解除原因 |
| 同步 | 布尔 | 否 | `true` | 是否同步到关联服务器 |

示例：`/风纪 解除禁言 用户:@小明 原因:已改正`

### 2.4 `/风纪 查询记录`
查询某用户的处罚记录（与 `/处罚` 共用记录，两边都能查到）。

示例：`/风纪 查询记录 用户:@小明`

## 3. 管理员用的命令 `/风纪配置`

> 仅管理员可用（服务器所有者 / Administrator / 管理身份组）。

### 3.1 `/风纪配置 身份组`
管理“哪些身份组能用 `/风纪`”。

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 操作 | 选择(add/remove/list) | 是 | 添加 / 移除 / 查看列表 |
| 身份组 | 身份组 | 否 | add/remove 时必填；list 不填 |

示例：
- `/风纪配置 身份组 操作:add 身份组:@风纪委员`
- `/风纪配置 身份组 操作:remove 身份组:@风纪委员`
- `/风纪配置 身份组 操作:list`

> 名单为空时，只有管理员能用 `/风纪`，其他人一律被拒。

### 3.2 `/风纪配置 上限`
设定风纪的禁言/警告最长时长（**至少填一项**）。

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| 禁言上限 | 文本 | 否 | 如 `2h`、`3d`（不超过 28 天） |
| 警告上限 | 文本 | 否 | 如 `7d`、`12h` |

示例：
- `/风纪配置 上限 禁言上限:2h 警告上限:7d`
- `/风纪配置 上限 禁言上限:6h`（只改禁言，警告保持原值）

### 3.3 `/风纪配置 查看`
查看当前风纪配置（可用身份组 + 两个上限）。

示例：`/风纪配置 查看`

---

## 4. 时间格式（同 `/处罚`）

- `12h` = 12 小时，`3d` = 3 天，`7` = 7 天（纯数字默认按天）
- 不能填 0 或负数
- 禁言最长 28 天（Discord 限制），风纪的上限也不会超过这个值

---

## 5. 常见问题

**Q：风纪委员提示“权限不足”？**
让管理员用 `/风纪配置 身份组 add` 把他的身份组加进名单。

**Q：提示“时长超过上限”？**
超过了管理员在 `/风纪配置 上限` 设的值，缩短时长或让管理员放宽上限。

**Q：警告命令失败？**
通常是没配警告身份组，管理员执行一次 `/处罚 配置警告身份组`。

**Q：公告发到哪、能不能跨服同步？**
和 `/处罚` 完全一样，用的是同一套公告频道与同步目标配置。
