# 分服受控邀请模块使用说明

---

## 1. 这个模块是干什么的？

一句话：

**让用户只能通过“自己申请的一次性邀请码”进入分服，防止邀请码乱传。**

它会自动处理这些事：
- 用户点按钮申请邀请码
- 检查资格（身份组、冷却、黑名单）
- 生成一次性邀请码（1 次使用、限时）
- 监控入服行为（非法加入可踢/封）
- 记录日志（现在是 Embed 彩色日志）

---

## 2. 你需要先知道的概念

- **主服**：用户平时待着、点“申请邀请码”按钮的服务器
- **分服**：用户最终要加入的目标服务器
- **受控邀请入口**：一条带按钮的消息，用户点按钮来申请邀请码

---

## 3. 开始前准备（必看）

请确保 Bot：
1. 已加入主服和分服
2. 在分服有这些权限：
   - `CreateInstantInvite`（创建邀请）
   - `ManageGuild`
   - `BanMembers`
   - `KickMembers`
3. 能读取你指定的频道（入口频道、日志频道、分服邀请码频道）

---

## 4. 最快配置流程（推荐顺序）

> 以下命令都在主服执行。

### 第 1 步：绑定主服和分服

```bash
/分服受控邀请 绑定 主服务器id:<主服ID> 分服务器id:<分服ID>
```

---

### 第 2 步：设置分服的邀请码创建频道

```bash
/分服受控邀请 设置邀请码频道 分服务器id:<分服ID> 频道id:<分服文字频道ID>
```

这个频道是 Bot 在分服里真正创建邀请码的地方。

---

### 第 3 步：创建入口消息（用户点击申请）

```bash
/分服受控邀请 创建入口 频道:<主服中的某个频道>
```

执行后会在指定频道发出“申请邀请码”入口消息。

---

### 第 4 步（可选）：设置资格身份组

如果你希望“只有某些人能申请”：

```bash
/分服受控邀请 资格角色添加 身份组:<主服身份组>
```

如果不设置资格组，默认是主服成员都可以申请（仍受黑名单/冷却限制）。

---

### 第 5 步（可选）：设置日志频道

```bash
/分服受控邀请 设置日志频道 频道id:<日志频道ID> 分服务器id:<分服ID>
```

现在日志是 **Embed 彩色**：
- 绿色：正常/成功
- 黄色：警告/可疑
- 红色：严重异常/违规处理
- 蓝色：一般信息

---

## 5. 普通用户查看自己状态

你现在有两个入口（功能一样）：

1. 独立命令（推荐）
```bash
/查看我的分服受控邀请状态
```

2. 原子命令（保留兼容）
```bash
/分服受控邀请 我的状态
```

会显示：
- 是否在黑名单
- 各分服冷却状态
- 当前活跃邀请码

## 7. 多分服怎么工作？

当前设计是：

- 一个入口消息里放多个按钮（每个分服一个按钮）
- 用户点哪个按钮，就申请哪个分服的邀请码

注意：单条按钮行上限是 5 个按钮（Discord 组件限制）。

---

## 8. 推荐上线自测清单

上线前建议用小号测一次完整流程：

1. 小号在主服，有资格组
2. 点入口按钮，拿到邀请码
3. 用邀请码进分服
4. 看日志频道是否出现对应日志
5. 再测一次冷却是否生效
6. 测试黑名单用户是否被拦截

## 9. 常见问题（FAQ）

### Q1：用户点按钮提示“缺少邀请码频道”
说明该分服还没配置 `/设置邀请码频道`。

### Q2：看不到新命令
重启 Bot，确保命令已重新注册到目标服务器（`GUILD_IDS` 配置正确）。

### Q3：想让“查看我的状态”只给部分人看到
用独立命令 `/查看我的分服受控邀请状态` 去做命令权限控制；原子命令可保留兼容。

### Q4：入口内容参数改了但消息没变
执行：
```bash
/分服受控邀请 刷新入口
```

---

## 10. 给管理员的实用建议

- 先开日志频道，再开放入口，出问题更容易排查
- 不要把分服普通邀请链接公开在别处
- 定期看 `活跃邀请码` 和 `黑名单列表`
- 多分服时，尽量统一参数（有效期/冷却）避免用户困惑
